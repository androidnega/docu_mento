DOCU MENTO – SYSTEM & DATA STRUCTURE
====================================

This document summarizes the current high-level structure of the Docu Mento application based on the codebase in this workspace (models, migrations, and seeders).

----------------------------------------------------------------------
1. CORE DOMAINS / MODULES
----------------------------------------------------------------------

The system is organized around several main domains:

- Core Institutions & Users
- Academic Structure (levels, semesters, courses, classes, class groups)
- Students & Authentication (indices, OTP, WebAuthn passkeys, push subscriptions)
- Quiz / Exam Flow (class groups, courses, exam calendar, attendance uploads)
- Docu Mentor (projects, groups, categories, chapters, submissions, AI reviews, scores)
- SMS Logging
- Miscellaneous Documents

Below each section lists the key models and their implied tables/relations.

----------------------------------------------------------------------
2. CORE INSTITUTION & USER STRUCTURE
----------------------------------------------------------------------

2.1 Schools, Institutions, Faculties, Departments
----------------------------------------

- schools
  - Model: App\Models\School
  - Columns:
    - id (bigint, PK)
    - name (string, unique)
    - is_active (bool, default true)
    - created_at, updated_at
  - Relations:
    - hasMany departments

- institutions
  - Model: App\Models\Institution
  - Columns (from model fillable & usage):
    - id (bigint, PK)
    - name (string)
    - region (string, nullable)
    - logo (string, nullable)
  - Relations:
    - hasMany users (to users.id)
    - hasMany faculties (to faculties.institution_id)

- faculties
  - Model: App\Models\Faculty
  - Columns:
    - id (bigint, PK)
    - institution_id (FK → institutions.id)
    - name (string)
  - Relations:
    - belongsTo institution
    - hasMany departments
    - hasMany users

- departments
  - Model: App\Models\Department
  - Columns:
    - id (bigint, PK)
    - faculty_id (FK → faculties.id)
    - school_id (FK → schools.id, nullable)
    - name (string)
    - is_active (bool, default true)
  - Relations:
    - belongsTo faculty
    - belongsTo school
    - hasMany users

2.2 Roles, Users & Identity
---------------------------

- roles
  - Model: App\Models\Role
  - Columns:
    - id (bigint, PK)
    - department_id (FK → departments.id, cascade on delete)
    - name (string)
    - created_at, updated_at
    - unique (department_id, name)
  - Relations:
    - belongsTo department
    - hasMany users

- users
  - Model: App\Models\User (extends Authenticatable)
  - Key columns (from $fillable and casts):
    - id (bigint, PK)
    - username (string, unique)
    - email (string, unique, nullable for some users)
    - phone (string, nullable)
    - index_number (string, nullable; used to link to Student)
    - name (string)
    - course_id (FK → courses.id, nullable)
    - role (string)
      - Core roles:
        - super_admin
        - examiner
      - Docu Mentor roles:
        - student
        - leader
        - supervisor
        - hod
        - coordinator
    - role_id (FK → roles.id, nullable; restrictOnDelete)
    - is_active (bool, default true)
    - password (hashed)
    - avatar (string, nullable)
    - institution_id (FK → institutions.id, nullable)
    - faculty_id (FK → faculties.id, nullable)
    - department_id (FK → departments.id, nullable)
    - sms_allocation (int, nullable)
    - sms_used (int, nullable)
    - ai_quiz_tokens_allocation (int, nullable)
    - ai_quiz_tokens_used (int, nullable)
    - ai_quiz_tokens_reset_at (datetime, nullable)
    - group_leader (bool)
    - coordinator (bool)
    - remember_token, created_at, updated_at
  - Relations:
    - belongsTo institution, faculty, department, course
    - belongsTo Role via role_id (roleModel)
    - hasMany Contact (contacts)
    - hasOne Supervisor (supervisorProfile)
    - belongsToMany courses via course_user (examiner assignments)
    - hasMany classGroups (ClassGroup.examiner_id)
    - belongsToMany classGroupsTeaching via class_group_course (examiner per course)
    - belongsToMany DocuMentor\ProjectGroup via group_members (docuMentorGroups)
    - hasMany DocuMentor\ProjectGroup via leader_id (ledDocuMentorGroups)
    - belongsToMany DocuMentor\Project via project_supervisors (supervisedProjects)
    - other helper scopes for coordinator/examiner visibility

- contacts
  - Model: App\Models\Contact
  - Columns:
    - id (bigint, PK)
    - user_id (unsigned bigint, cascade on delete in app)
    - phone (string)
    - created_at, updated_at
    - unique (user_id, phone)
  - Relations:
    - belongsTo user
    - hasMany SmsLog (smsLogs); cascade delete logs when contact deleted (booted)

- supervisors
  - Model: App\Models\Supervisor
  - Columns:
    - id (bigint, PK)
    - user_id (FK → users.id, unique, cascade on delete)
    - created_at, updated_at
  - Relations:
    - belongsTo user

2.3 Settings & Staff Password Resets
------------------------------------

- settings
  - Model: App\Models\Setting
  - Stores global application settings (table definition in migrations outside this workspace snapshot).

- staff_password_resets
  - Model: App\Models\StaffPasswordReset
  - Used for staff password reset flows (email + token).

----------------------------------------------------------------------

3.1 Levels and Semesters
------------------------

- student_levels
  - Model: App\Models\StudentLevel
  - Columns:
    - id (bigint, PK)
    - value (int; e.g. 100, 200, 300, 400)
    - label (string; display label)
    - sort_order (int)

- semesters
  - Model: App\Models\Semester
  - Columns:
    - id (bigint, PK)
    - value (int; e.g. 1, 2)
    - name (string)
    - sort_order (int)
  - Relations:
    - hasMany courses
    - hasMany students

3.2 Courses
-----------

- courses
  - Model: App\Models\Course
  - Columns:
    - id (bigint, PK)
    - name (string; stored in UPPERCASE)
    - code (string, unique)
    - is_archived (bool, default false)
    - quiz_category_id (FK → quiz_categories.id)
    - level_id (FK → student_levels.id)
    - semester_id (FK → semesters.id)
  - Relations:
    - belongsTo QuizCategory
    - belongsTo StudentLevel (level)
    - belongsTo Semester
    - hasMany ValidIndex (valid_indices)
    - belongsToMany users via course_user (examiners)
    - belongsToMany class_groups via class_group_course

3.3 Academic Classes & Class Groups
-----------------------------------

- academic_classes
  - Model: App\Models\AcademicClass
  - Columns:
    - id (bigint, PK)
    - name (string; e.g. "BTECH IT Level 100")
    - quiz_category_id (FK → quiz_categories.id)
    - level_id (FK → student_levels.id)
    - academic_year_id (FK → docu_mentor.academic_years.id)
  - Relations:
    - belongsTo QuizCategory
    - belongsTo StudentLevel
    - belongsTo DocuMentor\AcademicYear
    - hasMany students
    - hasMany class_groups

- class_groups
  - Model: App\Models\ClassGroup
  - Columns (from $fillable and constants):
    - id (bigint, PK)
    - name (string)
    - examiner_id (FK → users.id)
    - quiz_category_id (FK → quiz_categories.id)
    - level_id (FK → student_levels.id)
    - semester_id (FK → semesters.id, nullable)
    - academic_year_id (FK → docu_mentor.academic_years.id, nullable)
    - academic_class_id (FK → academic_classes.id, nullable)
    - accent_color (string, nullable; one of predefined keys)
    - allowed_devices (string; values: desktop, mobile, both)
    - created_at, updated_at
  - Relations:
    - belongsTo examiner (User)
    - belongsTo quizCategory, level, semester, academicYear, academicClass
    - hasMany ClassGroupStudent (students)
    - belongsToMany Course via class_group_course (with examiner_id on pivot)
    - hasMany ExamCalendar

- class_group_reps
  - Migration: 2026_02_21_100000_create_class_group_reps_table
  - Columns:
    - id (bigint, PK)
    - class_group_id (FK → class_groups.id, cascade on delete)
    - user_id (FK → users.id, cascade on delete)
    - created_at, updated_at
    - unique (class_group_id, user_id)

- class_group_students
  - Model: App\Models\ClassGroupStudent
  - Columns:
    - id (bigint, PK)
    - class_group_id (FK → class_groups.id)
    - index_number (string)
    - student_name (string)
  - Relations:
    - belongsTo ClassGroup
    - belongsTo Student (studentAccount) by index_number

3.4 Valid Indices

  - Columns:

7. NOTIFICATION & COMMUNICATION TRACKING (SMS)
----------------------------------------------------------------------
  Structure: Contact → SMS Log → Status.
  restrictOnDelete for status (app-level); cascade on contact delete (app-level).
  Store response, track delivery status, retry logic by status.

- sms_statuses
  - Model: App\Models\SmsStatus
  - Columns:
    - id (bigint, PK)
    - name (string, unique)
  - Relations:
    - hasMany SmsLog (status_id)
  - Helpers: isSuccess(), shouldRetry(). restrictOnDelete in booted().

- sms_logs
  - Model: App\Models\SmsLog
  - Columns:
    - id (bigint, PK)
    - status_id (unsigned bigint, index) — references sms_statuses
    - contact_id (unsigned bigint, nullable, index) — references contacts
    - message (text)
    - response (text, nullable)
    - user_id (unsigned bigint, nullable, index)
    - phone (string, nullable; legacy)
    - status (string, nullable; legacy)
    - created_at, updated_at
  - Relations:
    - belongsTo Contact (contact), SmsStatus (statusRef), User (user)
  - Helpers: logSend() (legacy), logWithStatus() (status_id + contact_id). Cascade delete when Contact deleted (Contact::booted).
